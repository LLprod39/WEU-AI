# Subagents (под-агенты) Cursor

Под-агенты — это отдельные ассистенты, которым основной агент Cursor может делегировать задачи. У каждого свой контекст, что удобно для длинных исследований или параллельной работы.

## Где лежат

- **Проект:** `.cursor/agents/` — только для этого репозитория.
- **Пользователь:** `~/.cursor/agents/` — для всех проектов.

Файлы — обычный Markdown с YAML frontmatter в начале.

## Формат файла

```markdown
---
name: Название под-агента
description: Краткое описание (видно в списке инструментов агента)
---

# Заголовок

Текст промпта: что должен делать под-агент, в каком формате отвечать, какие ограничения соблюдать.
```

Опционально в frontmatter (если Cursor поддерживает в твоей версии):

- `model` — конкретная модель (например `claude-3-sonnet-20240229`), не `auto`.
- `tools` — список инструментов (например `codebase_search`, `run_terminal_cmd`).
- `readonly` — `true` / `false`, может ли под-агент менять файлы.

## Пример в этом проекте

- **verifier.md** — проверяет выполненную работу: запускает тесты, проверяет работоспособность, возвращает отчёт «что прошло / что не доделано».

## Как использовать

1. Положи `.md` в `.cursor/agents/` (или в `~/.cursor/agents/`).
2. В чате с агентом можно сказать: «Дай на проверку под-агенту Verifier» или «Запусти Verifier для этой задачи».
3. Список доступных под-агентов Cursor берёт из содержимого `.cursor/agents/`.

# Subagents в проекте WEU

## Что такое subagent

**Subagent** в Cursor — это отдельный ассистент с собственным контекстом. Основной агент может делегировать ему задачу (например, «проверь этот код» или «исследуй кодовую базу»), под-агент выполняет её и возвращает результат.

- **Изоляция контекста** — длинные исследования не забивают основной чат.
- **Параллельная работа** — можно запускать несколько под-агентов одновременно.
- **Специализация** — у каждого свой промпт и (опционально) модель/инструменты.

## Где настроены под-агенты в проекте

Папка: **`.cursor/agents/`**

Сейчас там:

| Файл        | Назначение |
|------------|------------|
| `verifier.md` | Проверка выполненной работы: тесты, работоспособность, отчёт «что прошло / что не доделано». |
| `README.md`   | Краткое описание формата и использования под-агентов. |

## Как добавить свой subagent

1. Создай в `.cursor/agents/` файл с расширением `.md`, например `security_auditor.md`.
2. В начале файла добавь YAML frontmatter:

   ```yaml
   ---
   name: Security Auditor
   description: Проверяет код на типичные уязвимости и OWASP Top 10
   ---
   ```

3. Ниже напиши промпт: что должен делать под-агент, в каком формате отвечать, какие ограничения соблюдать.
4. Сохрани файл — Cursor подхватит новый под-агент из `.cursor/agents/`.

Подробнее и примеры — в [.cursor/agents/README.md](../agents/README.md). Официальная документация: [Subagents | Cursor Docs](https://cursor.com/docs/context/subagents).
